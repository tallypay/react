# Security policy

## Supported versions

We support the latest published `0.x` minor on npm. Older `0.x` minors receive security fixes on a best-effort basis until the next minor lands.

## Reporting a vulnerability

**Please do not open a public issue for security reports.**

Email **lukedaviesdev@gmail.com** with:

- A description of the issue and its impact.
- Steps to reproduce, or a minimal proof-of-concept.
- Affected version(s), if known.
- Whether you've publicly disclosed any of this anywhere yet.

You'll get an acknowledgement within 72 hours and a more detailed response within 7 days. Critical issues are fixed and released as a patch version; we'll coordinate disclosure with you and credit you in the release notes (unless you'd rather stay anonymous).

## Scope

This policy covers `@tallypay/core` source and published npm artefacts. The hosted TallyPay dashboard / collector is a separate target — report those at the same address with `[hosted]` in the subject.
